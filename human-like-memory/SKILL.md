# Human-Like Memory

当此 Skill 被激活时，Agent应遵循以下指令。

## 何时使用

当用户输入任何内容，都应调用本 Skill。调用后：

- **若内容不包含用户主动输入的有意义内容**（如仅包含错误提示、复制内容、空消息）→ 根据**用户画像**回复，结束流程
- **若有意义的用户输入** → 继续执行以下记忆相关步骤

## 前提条件

此 Skill 需要以下工具：
- Bash（用于执行 Python 脚本）

## 核心概念

- **事件**：用户输入中包含的独立事件
- **三层记忆**：每个事件有三个精细程度的版本
  - **第一层**：最概括，3-5字短语（用于快速检索和去重）
  - **第二层**：中等，包含主体、动作、对象
  - **第三层**：最精细，完整原始描述
- **重要性等级**：1-10，重复提及+1，用户可手动调整，等级越高遗忘越慢
- **情绪状态**：从输入内容判断，包含情绪类型（高兴/悲伤/愤怒/焦虑/平静/兴奋等）和强度（1-10）
- **用户画像**：长期存储，包含性格特征、爱好偏好、生活习惯等

## 指令步骤

### 1. 拆分事件 & 情绪分析

分析用户输入，拆分为若干独立事件。每个事件提取：
- **主体**：谁在做这件事
- **动作**：做了什么
- **对象**：作用于谁/什么
- **时间**：用户提供的（可选），若未提供则自动记录当前时间，并标记 `time_source: "user"` 或 `"system"`
- **情绪**：从内容判断情绪类型（高兴/悲伤/愤怒/焦虑/平静/兴奋/恐惧/惊讶等）和强度（1-10）
- **原始描述**：用户的完整原话
- **重要性**：AI根据情绪强度、情感词、切身利益、人物关系等自动分析（1-10）

若用户输入包含多个事件，逐个处理。

### 1.5 提取用户画像

从输入中分析并提取：
- **性格特征**：如内向、外向、理性、感性、谨慎、冒险等
- **爱好偏好**：如音乐、阅读、运动、旅游、美食等
- **生活习惯**：如作息规律、饮食习惯、消费偏好等
- **价值观倾向**：如重视家庭、事业、友情、自由等

用户画像长期累加存储，旧信息不删除只新增/更新置信度。

### 2. 生成三层记忆

对每个事件生成三个版本：
- **第一层（概要key）**：如"买咖啡"、"开会"、"吵架"
- **第二层（结构化）**：`{主体, 动作, 对象, 时间}`
- **第三层（原始）**：完整原始描述

### 3. 读取已有记忆

```bash
python3 -c "from scripts.read_memory import read_synopsis; print(read_synopsis())"
```

返回所有事件的**第一层概要列表**。

### 4. 比对相似度

将拆分出的事件**第一层**与已有记忆比对：
- **完全匹配**：第一层相同 → 询问用户是"继续同一事件"还是"开启新事件"
- **部分相似**：有关联词（如"又"、"再"、"继续"）→ 视为同一事件的延续
- **全新事件**：无匹配 → 视为新事件

### 5. 存储记忆

**自动分析重要性**：
```bash
python3 -c "from scripts.forget import analyze_importance; importance = analyze_importance('用户买了杯美式', {'type': '平静', 'intensity': 5}); print(importance)"
```

**新事件**：
```bash
python3 -c "from scripts.memorise import store_event; store_event('买咖啡', {'level1': '买咖啡', 'level2': {'主体': '用户', '动作': '买', '对象': '咖啡', '时间': '2026-04-19 15:30', 'time_source': 'system'}, 'level3': '用户买了杯美式', '情绪': {'type': '平静', 'intensity': 5}, '重要性': 6, 'last_accessed': '2026-04-19 15:30'})"
```

- key：level1 概要
- value：包含完整 level1/level2/level3、情绪、自动分析的重要性、最后访问时间

**继续已有事件**：更新该事件的 level3（追加新描述），重要性+1

### 5.2 遗忘处理（每次回复前调用）

```bash
python3 -c "from scripts.forget import process_forgetting; process_forgetting()"
```

遗忘规则（基于艾宾浩斯遗忘曲线 + 重要性等级）：
- **L3遗忘**：重要性1-10分别对应3-60天后，原始描述简化为"..."模糊版本
- **L2部分遗忘**：重要性1-10分别对应14-180天后，主体/动作/对象逐个模糊化
- **L1全部遗忘**：重要性1-10分别对应30-365天后，整个事件从记忆库删除

重要性等级越高，遗忘周期越长。

### 5.5 更新用户画像

从本次输入中提取新的性格/爱好/习惯特征，更新到画像库：
```bash
python3 -c "from scripts.memorise import update_persona; update_persona({'性格': {'外向': 0.8}, '爱好': {'咖啡': 0.9}})"
```

### 6. 综合记忆与画像回复

综合**事件记忆**和**用户画像**生成回复：
- 结合用户的性格、爱好、习惯，调整语气和内容风格
- 根据情绪状态调整表达方式（如同情、兴奋、理性分析等）
- 主动关联相关记忆，给出更个性化的回应

告诉用户：
- 记忆了什么事件（展示第一层概要）
- 当前情绪状态（判断结果）
- 是否与已有记忆关联
- 当前记忆库中共有多少条记忆

### 7. 画像确认（可选）

如识别到新的性格/爱好/习惯，可询问用户确认：
- "我注意到你似乎喜欢喝咖啡，对吗？"
- "你今天心情看起来不错？"

## 数据存储

### 事件记忆 `data/memory.json`
```json
{
  "买咖啡": {
    "level1": "买咖啡",
    "level2": {
      "主体": "用户",
      "动作": "买",
      "对象": "咖啡",
      "时间": "2024-01-01 10:00",
      "time_source": "user"
    },
    "level3": "用户买了杯美式",
    "情绪": {
      "type": "平静",
      "intensity": 5
    },
    "重要性": 5,
    "last_accessed": "2026-04-19 15:30"
  }
}
```

### 用户画像 `data/persona.json`
```json
{
  "性格": {
    "外向": {"置信度": 0.85, "来源次数": 12},
    "理性": {"置信度": 0.7, "来源次数": 8}
  },
  "爱好": {
    "咖啡": {"置信度": 0.9, "来源次数": 15},
    "阅读": {"置信度": 0.75, "来源次数": 6}
  },
  "习惯": {
    "晚睡": {"置信度": 0.6, "来源次数": 4}
  }
}
```

置信度 = 来源次数 / 总分析次数，随数据累积动态调整。
